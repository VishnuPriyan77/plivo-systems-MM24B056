# NOTES

The sender uses a compact FEC protocol over the relay: every media frame is sent once as a 161-byte packet, each even/odd pair gets an adjacent XOR repair packet, and almost every pair also gets a diagonal XOR repair packet that links the current odd frame to a previous even frame. The diagonal repair replaces duplicate parity so clustered losses are handled better while staying under the 2.0x relay-byte budget. The receiver uses `T0` and the packet's 6-bit sequence residue to reconstruct the absolute frame number, stores frames and XOR equations in fixed arrays, runs a small local repair pass, and sends each recovered frame to the player once. I did not use retransmission because the request and response would both cross the lossy jittery relay and usually arrive too late at low playout delays. Grade at `--delay_ms 98`; `97 ms` failed a hard full B-profile seed and `90 ms` failed several full B-profile seeds. This design breaks when too many related data and repair packets in the same local equation graph are lost or late. It also assumes relay delay is well below the 64-frame residue wrap window, which is true for the provided profiles and the stress cases tested.
